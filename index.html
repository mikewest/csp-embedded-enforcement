Abstract

This document defines a mechanism by which a web page can embed a nested browsing context if and only if it agrees to enforce a particular set of restrictions upon itself.

1. Introduction

This section is not normative.

1.1. Examples

MegaCorp Inc. wishes to ensure that the advertisements that run on its various publications are locked down to include script from trusted origins that have been audited for safety. They can do so by including the advertisement via an iframe element with a csp attribute: 
<iframe src="https://advertisements-r-us.example.com/ad1.cfm"
        csp="script-src https://trusted-cdn.example.com/">
</iframe>

The advertisment will only load if it is delivered with a Content-Security-Policy HTTP response header which is at least as restrictive as the policy specified by the embedder.

2. Framework

2.1. Modifications to HTML

Browsing contexts have an iframe security policy attribute, which is null unless otherwise specified.

iframe elements have a csp attribute which specifies the policy that an embedded document must agree to enforce upon itself.

partial interface HTMLIFrameElement {
  attribute DOMString csp;
};

HTMLIFrameElement's csp IDL attribute reflects the value of the element’s csp content attribute.

When an iframe element with a csp attribute has its nested browsing context created (before the intial about:blank Document is created), and when an iframe element’s csp attribute is set or changed while it has a nested browsing context, the user agent must set the nested browsing context’s iframe security policy to the result of executing the parse the policy algorithm on the csp attribute’s value.

During the navigate algorithm, perform the following step after the current step 19. At this point, the user agent has fetched a response which it is about to begin parsing, and redirects have been processed:

  1. If the algorithm in §3.1 Is resource blocked by browsing context’s iframe security policy? returns Blocked when executed upon the resource and the browsing context being navigated, abort these steps. The user agent MAY indicate to the user that navigation has been aborted for security reasons.

3. Algorithms

3.1. Is resource blocked by browsing context’s iframe security policy?

Given a resource (resource) and a browsing context (context), this algorithm returns Allowed or Blocked as appropriate:

  1. Let embedder policy be the value of context’s iframe security policy.

  2. Let resource policy be the result of executing the parse the policy algorithm on the value of resource’s Content-Security-Policy HTTP response header field, or null if no such header exists.

  3. If the §3.2 Is policy A subsumed under policy B? algorithm returns Subsumed when executed upon resource policy and embedder policy (that is, if the resource’s policy is subsumed under the embedder’s policy), return Allowed.

  4. Return Blocked.

3.2. Is policy A subsumed under policy B?

One Content Security Policy (policy A) is said to be subsumed under another (policy B) if there is no case which would violate policy B which would not also violate policy A. Given two policies, this algorithm returns Subsumed or Not Subsumed as appropriate:

  1. If policy B is null, return Subsumed.

  2. If policy A is null, return Not Subsumed.

  3. For each subsuming directive in policy B’s list of directives:

    1. Switch on subsuming directive’s name, and execute the associated steps:

      default-src

      1. For each directive name in child-src, connect-src, font-src, img-src, media-src, object-src, script-src, and style-src:

        1. Let subsumee directive be the result of executing §3.3 Get the effective directive value for directive name in policy on directive name and policy A.

        2. If subsumee directive is Not Present, return Not Subsumed.

        3. If §3.4 Is source list A subsumed under source list B? returns Not Subsumed when executed upon subsumee directive and subsuming directive, return Not Subsumed.

      child-src

      connect-src

      font-src

      img-src

      media-src

      object-src

      script-src

      style-src

      1. Let subsumee directive be the result of executing §3.3 Get the effective directive value for directive name in policy on the specified directive name and policy A.

      2. If subsumee directive is Not Present, return Not Subsumed.

      3. If §3.4 Is source list A subsumed under source list B? returns Not Subsumed when executed upon subsumee directive and subsuming directive, return Not Subsumed.

      Other

      We’re ignoring these for the moment. That’s probably a reasonable choice for some directives, but not so wonderful for others.

  4. Return Subsumed.

3.3. Get the effective directive value for directive name in policy

  1. If directive name is present in policy, then return its value.

  2. If default-src is present in policy, then return its value.

  3. Return Not Present.

3.4. Is source list A subsumed under source list B?

  1. If source list A is empty, return Subsumed.

  2. If source list B is empty, return Not Subsumed.

  3. Magic happens here.

  4. Return Not Subsumed.

This algorithm needs a tiny bit of work around step 3.

Conformance

Conformance requirements are expressed with a combination of descriptive assertions and RFC 2119 terminology. The key words “MUST”, “MUST NOT”, “REQUIRED”, “SHALL”, “SHALL NOT”, “SHOULD”, “SHOULD NOT”, “RECOMMENDED”, “MAY”, and “OPTIONAL” in the normative parts of this document are to be interpreted as described in RFC 2119. However, for readability, these words do not appear in all uppercase letters in this specification.

All of the text of this specification is normative except sections explicitly marked as non-normative, examples, and notes. [RFC2119]

Examples in this specification are introduced with the words “for example” or are set apart from the normative text with class="example", like this:

This is an example of an informative example.

Informative notes begin with the word “Note” and are set apart from the normative text with class="note", like this:

Note, this is an informative note.
